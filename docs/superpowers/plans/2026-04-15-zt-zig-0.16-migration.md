# zt v0.7.0 — Zig 0.16.0 Migration Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate zt from Zig 0.15.2 to Zig 0.16.0 and cut release v0.7.0.

**Architecture:** Doc-based migration plan authored before 0.16.0 is installed locally. Execution starts once `zig version` reports `0.16.0+`. Strategy: replace `@cImport` with `addTranslateC` modules in `build.zig`, then iterate `zig build` failures top-down. zt is low-level (direct `std.posix` syscalls, no `std.io` / `std.fs` high-level APIs, no `std.Thread` sync primitives), so the blast radius is narrower than average projects.

**Tech Stack:** Zig 0.16.0, X11/XCB, Wayland, FreeType, FontConfig, xkbcommon, xcb-imdkit.

**Reference:** https://ziglang.org/download/0.16.0/release-notes.html

- `posix.execvpeZ` signature drift — zt calls it in pty.zig and main.zig.

## Chunk 1: Build system — @cImport → addTranslateC

The biggest 0.16.0 change for zt. Each `@cImport` block becomes a `translateC` step in `build.zig` exposed as a module, then imported from Zig source.

### Task 1: Establish translateC module for XCB/xkb

**Files:**
- Modify: `build.zig` (add translateC steps)
- Modify: `src/backend/x11.zig:5-12` (replace @cImport)

- [ ] **Step 1.1: Read 0.16.0 addTranslateC docs**

Run: `zig build-obj --help 2>&1 | grep -i translate` as a sanity check; consult the release-notes "Build System Changes" section for `addTranslateC` signature.

- [ ] **Step 1.2: Add an `x11_c` translateC step in `build.zig`**

Before the `exe` compile step in `build.zig`, add:

```zig
const x11_c = b.addTranslateC(.{
    .root_source_file = b.path("src/backend/x11_cimports.h"),
    .target = target,
    .optimize = optimize,
});
x11_c.addIncludePath(b.path("sysroot/usr/include"));
// Add system include paths as currently wired to the exe.
const x11_c_mod = x11_c.createModule();
```

Then expose to the main module: `exe.root_module.addImport("x11_c", x11_c_mod);`

- [ ] **Step 1.3: Create `src/backend/x11_cimports.h`**

```c
#include <xcb/xcb.h>
#include <xcb/shm.h>
#include <sys/shm.h>
#include <xcb-imdkit/imclient.h>
#include <xkbcommon/xkbcommon.h>
#include <xkbcommon/xkbcommon-x11.h>
```

- [ ] **Step 1.4: Replace @cImport in `src/backend/x11.zig`**

Change lines 5–12 from:

```zig
const c = @cImport({
    @cInclude("xcb/xcb.h");
    @cInclude("xcb/shm.h");
    @cInclude("sys/shm.h");
    @cInclude("xcb-imdkit/imclient.h");
    @cInclude("xkbcommon/xkbcommon.h");
    @cInclude("xkbcommon/xkbcommon-x11.h");
});
```

To:

```zig
const c = @import("x11_c");
```

- [ ] **Step 1.5: Build just the x11 backend**

Run: `zig build 2>&1 | head -80`
Expected: XCB-related "unresolved import" errors gone. If new errors reference C symbols, ensure include paths/linked libraries are identical to the old cImport configuration.

- [ ] **Step 1.6: Commit**

```bash
git add build.zig src/backend/x11_cimports.h src/backend/x11.zig
git commit -m "build: migrate x11 backend @cImport to addTranslateC"
```

## Rollback plan

If migration stalls for >2 days:
1. Keep `feature/zig-0.16-migration` branch.
2. Document the specific blocker in `docs/superpowers/specs/zt-zig-0.16-blocker.md`.
3. Stay on 0.15.x for daily driver until blocker resolves upstream.

## Notes for the executor

- **Do not** refactor surrounding code while fixing migration errors. This plan is deliberately scoped to "make it compile on 0.16" — behavioral refactors belong in separate tasks.
- **Do not** introduce new features. v0.7.0 is a compatibility release.
- **Do not** rewrite `std.posix` call sites to use higher-level `std.Io` APIs, even if tempting. Stay low-level; zt's perf profile depends on it.
- Commit per logical unit (one per task where possible). Frequent commits make bisection easy if a migration fix regresses behavior.
